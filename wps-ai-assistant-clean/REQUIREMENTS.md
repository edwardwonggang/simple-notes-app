# WPS AI 对话插件需求文档

## 1. 项目目标

在 WPS 中提供一个可长期维护的 AI 对话工具，支持在 Mac 和 Windows 环境中稳定运行，并满足 AI 对话、流式输出、文档写入、Markdown 渲染、配置管理等核心需求。

该项目应避免当前旧实现中“想到哪里写到哪里”的问题，采用清晰的模块划分和可维护的架构设计。

## 2. 使用场景

用户在 WPS 右侧打开 AI 对话工具，在对话框中输入问题后：

1. 用户问题会直接写入 WPS 编辑区
2. AI 回复以流式方式返回
3. AI 回复内容实时写入 WPS 编辑区
4. 写入过程中需要实时把 Markdown 格式转换为 WPS 可见样式

## 3. 核心功能需求

### 3.1 右侧 AI 对话工具

- 在 WPS 中提供右侧侧栏形式的 AI 对话工具
- 支持在 Mac 版 WPS 和 Windows 版 WPS 中运行
- 支持用户在输入框中输入问题并发送
- 支持停止生成

### 3.2 AI 流式回复

- AI 回复必须支持流式输出
- 流式回复过程中，WPS 编辑区需要实时更新
- 不能等全部完成后一次性写入
- 用户应能感知到“正在持续生成”的过程

### 3.3 问题与回答写入 WPS 编辑区

- 用户输入的问题要直接写入 WPS 编辑区
- AI 的回答要直接写入 WPS 编辑区
- AI 回答需要边生成边写入
- 写入位置应保持稳定、可预测
- 写入后应尽可能自动滚动到最新内容

### 3.4 Markdown 到 WPS 格式渲染

AI 输出的 Markdown 内容需要在写入 WPS 时实时转换为对应格式，至少包括：

- 标题
- 段落
- 无序列表
- 有序列表
- 引用
- 代码块
- 粗体
- 斜体
- 行内代码
- 分隔线

### 3.5 配置管理

插件设置页需要支持以下配置项：

- API Key
- API URL
- 模型 ID
- 代理地址（可选）
- 是否忽略 TLS 证书错误（如需要）

要求：

- 配置应支持本地持久化保存
- 重新打开插件后应自动加载
- 配置文件仅保存在本地，不上传 Git

### 3.6 代理支持

- 代理为可选项
- 用户可以手动填写代理 URL
- 需要兼容公司网络环境
- Windows 环境可考虑兼容系统代理

## 4. 输入与快捷键背景说明

### 4.1 用户当前设备背景

- 用户当前主要在 Mac 上使用该项目
- 但用户使用的是 Windows 键盘
- 用户已经在系统层面实现了 Windows 键盘快捷键到原生 Mac 按键的映射转换

### 4.2 快捷键要求

- 设置页中的原生复制、剪切、粘贴、全选必须稳定可用
- 不允许出现部分输入框可用、部分输入框不可用
- 不允许出现粘贴后卡顿或长时间无响应
- 不应依赖不稳定的浏览器级剪贴板 hack
